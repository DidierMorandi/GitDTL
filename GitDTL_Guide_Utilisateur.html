GitDTL — Guide utilisateur

Guide de gestion quotidienne d’un projet Git avec l’interface GitDTL.

GitTkinterWindows

Objectif

GitDTL permet de gérer les opérations Git courantes avec des boutons numérotés et des explications lisibles.

Choisir un projet

  1. Ouvrir GitDTL.
  2. Vérifier le dossier affiché.
  3. Utiliser **Changer de projet** si le dossier n’est pas le bon.
Si le dossier n’est pas encore un dépôt Git, GitDTL peut proposer de l’initialiser.

Routine Git conseillée

1. Lire l’état

Commencer par l’option d’état du projet pour savoir quoi faire.

2. Voir les modifications

Consulter les différences avant d’enregistrer ou de valider.

3. Enregistrer puis valider

Ajouter les fichiers pertinents, puis créer un commit avec un message clair.

Publier sur GitHub

Après un commit, l’option de publication envoie les changements vers le dépôt distant configuré. Si aucun remote n’existe, GitDTL demande l’adresse GitHub.

Créer une version

La création de version associe un commit de version et un tag. Cette action sert à marquer une étape stable du projet.

Consulter le journal

Le journal aide à comprendre ce qui a été lancé et à relire les erreurs éventuelles. Il peut être consulté depuis l’interface.
