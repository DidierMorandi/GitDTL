GitDTL — Manuel de référence

Référence descriptive de l’interface graphique Git de la suite DTL.

Type : application TkinterLangage : PythonVersion code : v1.0-3

Rôle

GitDTL encapsule les opérations Git courantes dans une interface graphique numérotée. Il vise une lecture guidée de l’état d’un dépôt et une traduction des actions Git en termes accessibles.

Interface

Style visuel

L’interface reprend le style terminal DEC VT100 de la suite NetDTL : fond noir, vert phosphore et police monospace.

Guidage visuel

Les actions suggérées peuvent être mises en évidence selon l’état du dépôt, afin d’indiquér la prochaîne étape logique.

Modèle Git couvert

DomaineDescription
ÉtatLecture de l’état local, fichiers modifiés, non suivis ou indexés.
HistoriqueConsultation des commits et tags.
SynchronisationGestion des échanges avec `origin` et des branches suivies.
VersionCréation de tags annotés et publication associée.
ScanDécouverte de dépôts Git sous un dossier racine.

Aide et système expert

`aide.md` contient des textes d’aide contextuelle personnalisables. `expert_git.md` fournit des règles permettant d’associér des fragments de messages Git à des conseils explicatifs.

Journal applicatif

Les actions Git et erreurs sont écrites dans `logs/gitdtl.log`. Le journal sert à conserver une trace locale des opérations demandées et des résultats retournés par Git.

Limites

  • GitDTL dépend de Git installé localement.
  • L’application ne remplace pas les fonctionnalités avancées d’un cliént Git complet.
  • Les conseils experts dépendent de la couverture des règles locales.
