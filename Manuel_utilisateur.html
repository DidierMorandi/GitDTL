GitDTL

Git simplifié pour les projets DTL

Principe

GitDTL est conçu pour les administrateurs système débutants. L'application évite le jargon Git et présente les opérations sous forme d'actions simples : ajouter, modifier, supprimer, valider, publier et créer une version stable.

Devise : "Git pour l'administrateur système débutant."

Lancer GitDTL

Placez-vous dans le dossier du projet Git, puis lancez :

python GitDTL.py

Le bouton Changer de projet permet de choisir un autre dossier.

Menu principal

  • État du projet : affiche les fichiers modifiés, les fichiers non suivis et si le projet est prêt à publier.
  • Voir les modifications : affiche les différences détectées par Git.
  • Ajouter un fichier : ajoute un nouveau fichier au prochain enregistrement Git.
  • Mettre à jour un fichier : enregistre une modification pour la prochaine validation.
  • Supprimer un fichier : supprime un fichier du dépôt après confirmation.
  • Valider les changements : cree une validation avec une description.
  • Publier sur GitHub : publie le travail après vérification de l'état du projet.
  • Créer une release stable : crée une validation, une étiquette de version et publie la version.
  • Historique : affiche les 30 dernières validations.
  • Synchroniser depuis GitHub : récupère les changements distants.
  • Diagnostic GitDTL : produit un rapport lisible du depot.
  • Lire le journal : affiche la boite noire de GitDTL.

Securite

GitDTL demande une confirmation avant toute suppression et avant la création d'une version stable. Avant chaque publication, il vérifie l'état du projet et avertit si des fichiers ne sont pas encore validés.

Journal GitDTL

Le journal est créé automatiquement dans :

logs\gitdtl.log

Il indique ce qui a été fait, quand cela a été fait, quelles commandes Git ont été exécutées et dans quel ordre. La fenêtre du journal permet d'actualiser, d'effacer après confirmation et d'exporter une copie horodatée.
